5. As 5 Ferramentas de Coaching

tools

🎯 Roda da Vida — RodaDaVida.jsx

Avaliação das 8 áreas de vida do cliente usando RadarChart interativo. Permite comparar estado atual vs. estado ideal.

RadarChart Assessment entity 8 dimensões
  • Carreira · Finanças · Saúde · Relacionamentos
  • Família · Lazer · Desenvolvimento · Espiritualidade
  • Slider 0–10 por área + score geral calculado
  • Histórico de sessões com evolução temporal

📌 Planejador de Metas — Planejador.jsx

Criação e acompanhamento de metas no formato SMART. Kanban de status com drag-and-drop.

BarChart Goal entity SMART
  • Status: Pendente → Em Andamento → Concluída
  • Prazo, responsável, indicadores de sucesso
  • Gráfico de metas por mês (BarChart)
  • Categorização por área de vida

📓 Diário de Reflexão — DiarioReflexao.jsx

Registro de reflexões diárias com indicador de humor, prompts guiados e busca por data.

LineChart JournalEntry entity humor tracking
  • Escala de humor 1–5 com emoji
  • Prompts de coaching pré-definidos
  • Evolução do humor em LineChart
  • Tags personalizáveis por entrada

📈 Progresso — AcompanhamentoProgresso.jsx

Dashboard de evolução do cliente com múltiplos indicadores e comparação entre períodos.

AreaChart PieChart Progress entity
  • AreaChart de evolução semanal/mensal
  • PieChart de distribuição de categorias
  • KPIs: sessões, metas, score médio
  • Exportar relatório em PDF (futuro)

📝 Avaliação de Cliente — AvaliacaoCliente.jsx

Questionários customizáveis com scoring automático, histórico de respostas e comparação entre avaliações.

RadarChart Assessment entity questionários scoring automático
Tipos de questão
Escala Likert (1–5), múltipla escolha, texto livre, sim/não
Templates prontos
Roda da Vida, Valores, Propósito de Vida, Autossabotagem
Histórico comparativo
Comparação entre avaliações no RadarChart com sobreposição de períodos

7. Camada de Estado e Dados

state management

React Query — Padrões por entidade

Q

useQuery (leitura)

Listagens com paginação e cache automático. Ex: useQuery(['clients', coachId], fetchClients)

M

useMutation (escrita)

Criação/edição com invalidação de cache. Usar onSuccess: () => queryClient.invalidateQueries

O

Optimistic updates

Para ações de UI imediata (toggle status meta, registro de humor) usar onMutate + rollback em onError

Context API — Dois contextos globais

AuthContext
  • user — objeto do coach autenticado
  • role — 'coach' | 'admin'
  • isLoading — estado de carregamento
  • logout() — Base44 auth.logout
ThemeContext
  • colorScheme — 'light' | 'dark'
  • toggleTheme() — persiste em localStorage
  • Gradiente: slate-50 via blue-50 to indigo-50

Custom Hooks — Padrão recomendado

useClients.js
export function useClients() {
const {user} = useAuth()
return useQuery({
queryKey: ['clients', user.id],
queryFn: () => fetchClients()
})
}
useGoals.js
export function useGoals(clientId) {
return useQuery({
queryKey: ['goals', clientId],
queryFn: () => fetchGoals(clientId),
enabled: !!clientId
})
}
useCreateGoal.js
export function useCreateGoal() {
const qc = useQueryClient()
return useMutation({
mutationFn: createGoal,
onSuccess: () => qc.invalidateQueries(['goals'])
})
}

9. Autenticação e Controle de Acesso

auth + rbac

Fluxo de Autenticação (Base44)

1

Login

Base44 gerencia JWT via auth.login(email, password)

2

Token persistido

Base44 SDK armazena token automaticamente. auth.getCurrentUser() na inicialização do App

3

AuthContext populado

User e role disponíveis em toda a árvore via useAuth()

4

ProtectedRoute

Verifica autenticação. Rota /admin valida adicionalmente role === 'admin'

Papéis (Roles)

🧑‍💼 coach
  • Acesso às suas ferramentas e clientes
  • Dashboard pessoal com suas métricas
  • Criação e edição de avaliações
  • Não vê o Painel Admin
🛡️ admin
  • Tudo do role coach
  • Acesso ao Painel Admin (/admin)
  • Visão de todos os coaches e clientes
  • Métricas globais da plataforma

10. Decisões Arquiteturais Chave (ADRs)

architecture decisions
ADR-01 — Co-localização de lógica por domínio Cada ferramenta é um módulo autocontido em pages/tools/. Componentes de UI genéricos ficam em components/ui/ e componentes de domínio em components/shared/. Isso facilita escalar novas ferramentas sem impacto lateral.
ADR-02 — React Query como única fonte de verdade de servidor Nenhum estado de servidor é duplicado no Context API. O Context serve apenas para estado de UI global (auth, tema). React Query gerencia cache, refetch e sincronização automática com o Base44 SDK.
ADR-03 — Layout responsivo por breakpoint, não por componente alternativo O Layout.jsx usa classes Tailwind (lg:flex hidden / lg:hidden) para alternar entre sidebar desktop e header+footer mobile. Um único componente de layout gerencia ambos os contextos, evitando duplicação de lógica de navegação.
ADR-04 — Recharts com tema centralizado Toda configuração visual do Recharts (cores, margens, tooltips) fica em lib/chartTheme.js e é importada por todos os gráficos. Mudanças visuais globais requerem alteração em apenas um arquivo.
ADR-05 — Controle de idioma no HTML, não no JS lang="pt-BR" no <html> e meta name="google" content="notranslate" bloqueiam tradução automática do navegador e Google Translate na camada de markup, sem necessidade de bibliotecas i18n externas.
